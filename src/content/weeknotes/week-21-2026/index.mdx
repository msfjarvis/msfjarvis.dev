## Reading

I think the book itself is great, but with so many distractions available I found myself struggling to read it regularly. My usual fare of Science Fiction works better here by having continual suspense drawing me in, so I started reading [The Knife of Never Letting Go](https://bookwyrm.social/user/msfjarvis/generatednote/11327057) instead of going for more non-fiction books. I'll see if I can piece together a book review for Could Should Might Don't, and definitely see what I can change to make non-fiction stick better. I can't have great books take me 2 months to go through 400 pages when I typically manage that in under 2 weeks.

## Random LLM experiments

Just wanted to have some place to dump info about what shenanigans I have been up to, so I'll probably just make this an ongoing section.

I've written previously about [switching to OpenAI Codex](/weeknotes/week-19-2026/) and the associated GPT-5 laziness problems that I had to deal with. In the two weeks since, I've had multiple incidents of the extension I wrote either not firing when it should or misfiring when it shouldn't so I decided to make it entirely too complicated. As I say [in the commit message](https://git.msfjarvis.dev/msfjarvis/my-pi/commit/b243e6caf15ae2bfd648a65a2143429e9da1eda7); what more RegEx can't solve, more AI definitely can. Now instead of putting every message through an impenetrably complicated RegEx, the extension instead pipes it through an impenetrably complicated text classifier LLM. This approach also hasn't been foolproof so far either, but iterating over the prompt ([here](https://git.msfjarvis.dev/msfjarvis/my-pi/commit/7347bf2b607ca71d8682e853f0db95844660422a) and [here](https://git.msfjarvis.dev/msfjarvis/my-pi/commit/59d79e9892ac6c04c9ee02a0e09daf2db300dc71)), has been helpful when paired with a proper test suite.

[AGENTS.md](https://agents.md/) files have been the de-facto way of giving LLMs persistent knowledge of codebases, but I find it somewhat annoying to have a bunch of them in my repositories. This week I've been looking at alternatives, and unfortunately mostly came up empty. Only one thing stood out to me, which was [agentmemory](https://github.com/rohitg00/agentmemory). So far I've only put "memories" into it, and I'll see if during the next week LLMs actually use this or continue reading all files from scratch.

I've been going all in on [skills](https://agentskills.io) as a way of steering LLMs, and found an adversarial reviewer skill from my colleague [Matt Kane](https://mk.gg/) in the [EmDash CMS repo](https://github.com/emdash-cms/emdash/blob/283bcf059e8c6889d14292e822953f13eaf6a6b0/skills/adversarial-reviewer/SKILL.md). I had it run across the full diff of [Claw](https://github.com/msfjarvis/compose-lobsters) since the last release, and it was able to identify [a couple of valid problems](https://git.msfjarvis.dev/msfjarvis/compose-lobsters/commit/1b6ae68e0d4a27d99709a09b7ef759f6bcda5b17). Obviously like any LLM usage, it also fully made up how to properly detect login sessions but that wasn't too difficult to rectify early on.
